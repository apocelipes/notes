可以通过魔术方法实现运算符重载。

一些运算符有左右两个版本，比如`__add__`和`__radd__`，对应`a + other`和`other + a`。

调用规则，对于表达式`a + b`：

1. 如果b是a的子类型，则优先尝试`b.__radd__(a)`
2. 否则优先尝试`a.__add__(b)`
3. 优先尝试的方法如果不存在或者返回了`NotImplemented`，则尝试相对的左/右版本的运算符，比如`b.__radd__(a)`
4. 再次尝试失败则抛出`TypeError`
5. 两边操作数类型相同时永远只会调用`__add__`
6. eq和ne，则会调用另一个参数的同名方法。子类的方法也一样会被优先调用
7. ne如果没定义，则会用`not a.__eq__(b)`，如果最终eq和ne都没有合适的，eq会返回False，ne会返回True

为了兼容性和灵活性，重载的运算符应该先检查类型，对于不支持的操作直接返回`NotImplemented`。

对于表达式`if a`或`bool(a)`:

1. 如果重载了`__bool__`，则调用`__bool__`
2. 没有定义（不可以返回`NotImplemented`），则回退至`a.__len__() != 0`，
3. 没有定义len，则简单地认为对象是True
