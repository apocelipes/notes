宏体学名替换列表。两个宏从名字、形参（包括名字）到替换列表都完全相同，则认为它们完全等价，可以重复定义。

宏展开步骤：

1. 预处理器识别到宏，开始处理，并把当前宏加入禁止展开列表
2. 如果有参数，则先展开所有实参
3. 如果形参出现在`#`之后或者`##`两边，则不展开实参，直接把实参的原始文本填充进替换列表
4. 将替换完成的实参代入形参所在位置，然后处理`#`和`##`，两种直接没有优先级顺序规定，实际上clang和gcc的处理顺序就不同，因此最好不要在同一个替换列表里混用
5. 替换完成的宏体中如果还有别的宏（包括拼接替换处理的），则递归展开这些宏
6. 如果宏在禁止列表里出现，则不会展开，宏名作为普通token，这是为了防止无限递归
7. 所有处理完成后展开结束，把当前宏从禁止列表里移除。
8. 展开的内容是预处理指令时，这些内容只会被当作普通token而不是指令。

例子1：

```c
#define STR(x) #x
#define PI 3.14

STR(PI)  // "PI"
```

形参出现在`#`之后，所以实参不会展开，文本被原样转换成字符串。

例子2：

```c
#define A(x) x B
#define B A(2)

A(1) // "1 A(2)"
```

识别到A，实参不是宏不用展开，替换成`1 B`；

没有`#`和`##`因此进行下一步扫描，发现宏B；

B没有参数，直接展开，变成`A(2)`；

A在禁止列表里，因此不展开，B展开完成，最后整体变为`1 A(2)`

例子3：

双层宏惯用法：

```c
#define _str(s) #s
#define STR(s) _str(s)

#define A(x) x B
#define B A(2)

STR(A(1)) // "1 A(2)"
```

如果直接用`#arg`，则实参里的宏不会被展开，但很多适合我们是希望实参被展开的，因此我们利用展开顺序。

先用一个替换列表里形参不跟随`#`的宏，这样实参可以被展开，然后让展开内容做另一个转换字符串宏的参数。

STR会被替换成`_str(1 A(2))`，`_str`的形参在`#`之后，因此不展开直接原样转成字符串字面量。

例子4：

```c
#define X   100
#define Y   X
#define STR(a)  #a
#define CAT(a,b) a##b
#define WRAP(a) STR(CAT(a, Y))

WRAP(X) // "CAT(100, Y)"
```

WRAP先处理，实参展开成100，替换列表变为`STR(CAT(100, Y))`。

下面一步容易误解，实际上先处理STR而不是CAT，因此发现形参在`#`之后，实参不展开，变成了`CAT(100, Y)`。

例子5：

```c
#define C(a, b)   C_expanded(a, b)
#define B(x)      B_expanded(x)
#define A(y)      A_expanded(y)

#define ARG1 arg2
#define ARG2 arg1

STR(A(B(C(ARG1, ARG2)))) // "A_expanded(B_expanded(C_expanded(arg2, arg1)))"
```

预处理器处理的顺序是A->B->C，但因为实参需要展开，所以先从最内层的C展开。C的实参也是宏，先展开C的实参。

所以先变成`A(B(C_expanded(arg2, arg1)))`，然后是`A(B_expanded(C_expanded(arg2, arg1)))`，最后变成`A_expanded(B_expanded(C_expanded(arg2, arg1)))`。

总结：最外层先处理，最内层先展开。

当然如果中间有任何一层形参被`#`或者`##`处理，则展开会停止，比如B的形参在`#`之后，那C就不会被处理，也不会被展开。

`#include`后面可以跟宏，这样写是没问题的：

```c
#define _s(s) #s
#define SS(s) _s(s)

#define A stdio.h

#include SS(A)

#define B "stdint.h"

#include A
#include B
```

`SS(A)`和B会被替换，头文件可以正常引入，但这些不行：

```c
#define C stdio.h

#include <C>
#include "C"
```

这些都是预处理错误。
